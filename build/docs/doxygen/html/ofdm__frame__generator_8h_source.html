30  /*!
31  * \brief Create frequency domain OFDM symbols from complex values, add pilots.
32  * \ingroup mimo_ofdm_jrc
33  * \ingroup ofdm_blk
34  *
35  * \details
36  * This block turns a stream of complex, scalar modulation symbols into vectors
37  * which are the input for an IFFT in an OFDM transmitter. It also supports the
38  * possibility of placing pilot symbols onto the carriers.
39  *
40  * The carriers can be allocated freely, if a carrier is not allocated, it is set
41  * to zero. This allows doing OFDMA-style carrier allocations.
42  *
43  * Input: A tagged stream of complex scalars. The first item must have a tag
44  * containing the number of complex symbols in this frame.
45  * Output: A tagged stream of complex vectors of length fft_len. This can directly
46  * be connected to an FFT block. Make sure to set this block to 'reverse'
47  * for the IFFT. If \p output_is_shifted is true, the FFT block must activate
48  * FFT shifting, otherwise, set shifting to false. If given, sync words are
49  * prepended to the output. Note that sync words are prepended verbatim,
50  * make sure they are shifted (or not).
51  *
52  * Carrier indexes are always such that index 0 is the DC carrier (note: you should
53  * not allocate this carrier). The carriers below the DC carrier are either indexed
54  * with negative numbers, or with indexes larger than \p fft_len/2. Index -1 and index
55  * \p fft_len-1 both identify the carrier below the DC carrier.
56  *
57  * There are some basic checks in place during initialization which check that the
58  * carrier allocation table is valid. However, it is possible to overwrite data symbols
59  * with pilot symbols, or provide a carrier allocation that has mismatching pilot symbol
60  * positions and -values.
61  *
62  * Tags are propagated such that a tag on an incoming complex symbol is mapped to the
63  * corresponding OFDM symbol. There is one exception: If a tag is on the first OFDM
64  * symbol, it is assumed that this tag should stay there, so it is moved to the front
65  * even if a sync word is included (any other tags will never be attached to the
66  * sync word). This allows tags to control the transmit timing to pass through in the
67  * correct position.
68  */
